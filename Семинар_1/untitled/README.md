# Система Управления Спутниками (ООП)

Этот проект представляет собой реализацию консольной системы управления спутниковой группировкой. Проект разработан для демонстрации основных принципов объектно-ориентированного программирования (ООП) на языке Java.

## Описание проекта

Система моделирует работу космической группировки, состоящей из различных типов спутников:

- **Спутники связи (CommunicationSatellite)** - отвечают за передачу данных с определенной пропускной способностью.
- **Спутники дистанционного зондирования Земли (ImagingSatellite, ДЗЗ)** - занимаются фотосъемкой поверхности Земли с заданным разрешением.

Все спутники объединяются в **Спутниковую группировку (SatelliteConstellation)**, которая позволяет централизованно управлять ими (добавлять, активировать, запускать миссии).

## Принципы ООП в проекте

В коде наглядно представлены четыре основных принципа ООП:

1.  **Абстракция:** Выделение общих характеристик всех спутников в единый базовый класс `Satellite`. Использование абстрактного метода `performMission()`, который определяет контракт для выполнения миссии, но не ее конкретную реализацию.
2.  **Инкапсуляция:** Сокрытие внутреннего состояния объектов. Поля классов (`name`, `batteryLevel`, `bandwidth`, `resolution`) защищены модификаторами доступа (`private` или `protected`). Доступ к ним (при необходимости) осуществляется через методы (например, `getBandwidth()`, `getResolution()`). Логика расхода батареи инкапсулирована в методе `consumeBattery()`.
3.  **Наследование:** Классы `CommunicationSatellite` и `ImagingSatellite` расширяют базовый класс `Satellite`, наследуя его общие свойства (имя, заряд батареи) и методы (активация, деактивация), дополняя их своими специфичными (например, полоса пропускания или счетчик фото).
4.  **Полиморфизм:** Группировка `SatelliteConstellation` хранит все спутники в едином списке `List<Satellite>`. При вызове метода `executeAllMissions()` у группировки, она проходится по этому списку, вызывая метод `performMission()` у каждого спутника. Благодаря полиморфизму, каждый спутник по-разному реагирует на этот вызов (спутник связи передает данные, а ДЗЗ - делает снимок) в зависимости от своего реального типа объекта.

Также проект демонстрирует **агрегацию** — класс `SatelliteConstellation` содержит (агрегирует) объекты типа `Satellite`, но спутники могут существовать и независимо от группировки.

## Как запустить

1. Откройте проект в вашей любимой IDE (например, IntelliJ IDEA).
2. Перейдите в класс `Main.java` (находится в `src/main/java/org/example/Main.java`).
3. Запустите метод `main`.
4. В консоли отобразится пошаговый лог работы: создание спутников, формирование группировки, активация и выполнение миссий.
